# Cron Jobs для автоматического обновления кэшей

## Обзор

Файл `cron.ts` содержит Deno cron jobs для автоматического запуска задач обновления кэшей по расписанию.

## Запуск

```bash
deno task cron
```

## Расписание

| Таймфрейм | Расписание | Описание |
|-----------|------------|----------|
| **1h** | `0 * * * *` | Каждый час в :00 минут |
| **4h** | `0 */4 * * *` | Каждые 4 часа в :00 минут |
| **8h** | `0 */8 * * *` | Каждые 8 часов в :00 минут |
| **12h** | `0 */12 * * *` | Каждые 12 часов в :00 минут |
| **1d** | `0 0 * * *` | Ежедневно в 00:00 |

## Как это работает

1. **Загрузка переменных окружения**: Читает `SECRET_TOKEN` и `NGROK_URL` из `.env`
2. **Триггер задач**: Отправляет POST запросы на `{NGROK_URL}/api/jobs/run/{jobName}`
3. **Аутентификация**: Использует Bearer token из `SECRET_TOKEN`
4. **Логирование**: Выводит статус каждого запуска задачи

## Требования

- `NGROK_URL` должен быть установлен в `.env`
- `SECRET_TOKEN` должен быть установлен в `.env`
- Сервер должен быть запущен (`deno task start`)

## Примечания

- Все задачи выполняются **без задержек** - точно по расписанию
- Cron service работает независимо от основного сервера
- Для остановки используйте `Ctrl+C`
